# 关爱计划功能测试文档

## 一、测试概述

### 1.1 测试目的
验证关爱计划模块的完整功能，确保家属、管理员、志愿者三方角色能够正常使用各自的功能。

### 1.2 测试范围
- 家属端：发布计划、查看计划、查看服务记录
- 管理员端：审核计划、审核服务记录
- 志愿者端：接单大厅、认领计划、提交服务记录

### 1.3 测试环境
- 操作系统：Windows 10/11
- 浏览器：Chrome 90+、Firefox 85+、Edge 90+
- 后端：Java 8+、Tomcat 8.5+、MySQL 5.7+
- 框架：Spring MVC + MyBatis

## 四、接口测试用例

### 4.1 关爱计划接口

#### API-001: 创建关爱计划
```
POST /family/care-plan/create
Content-Type: application/x-www-form-urlencoded

参数：
- elderlyId: 关爱对象ID
- planName: 计划名称
- serviceContent: 服务内容
- totalPeriods: 总周期数
- servicesPerPeriod: 每周期服务次数

预期响应：
{
    "success": true,
    "message": "计划创建成功，等待审核"
}
```

#### API-002: 审核关爱计划
```
POST /admin/care-plan/audit/{planId}
Content-Type: application/x-www-form-urlencoded

参数：
- status: APPROVED/REJECTED
- remark: 审核备注（拒绝时必填）

预期响应：
{
    "success": true,
    "message": "审核完成"
}
```

#### API-003: 认领关爱计划
```
POST /volunteer/care-plan/claim/{planId}

预期响应：
{
    "success": true,
    "message": "认领成功"
}
```

### 4.2 服务记录接口

#### API-004: 提交服务记录
```
POST /volunteer/care-plan/record/submit
Content-Type: multipart/form-data

参数：
- planId: 计划ID
- serviceDate: 服务日期
- serviceTimeStart: 开始时间
- serviceTimeEnd: 结束时间
- serviceContent: 服务内容
- elderlyCondition: 关爱对象状况
- photos: 服务照片（多文件）

预期响应：
{
    "success": true,
    "message": "服务记录提交成功"
}
```

#### API-005: 审核服务记录-通过
```
POST /admin/care-plan/record/approve/{recordId}

预期响应：
{
    "success": true,
    "message": "审核通过"
}
```

#### API-006: 审核服务记录-拒绝
```
POST /admin/care-plan/record/reject/{recordId}
Content-Type: application/x-www-form-urlencoded

参数：
- remark: 拒绝原因

预期响应：
{
    "success": true,
    "message": "已拒绝"
}
```
