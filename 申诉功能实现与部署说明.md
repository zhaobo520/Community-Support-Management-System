## 需要执行的部署步骤

### 步骤 1：创建数据库表

在 MySQL 数据库 `community_help` 中执行以下 SQL：

```sql
USE community_help;

CREATE TABLE IF NOT EXISTS t_appeal (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '申诉ID',
  user_id BIGINT COMMENT '申诉用户ID',
  user_role VARCHAR(50) COMMENT '用户角色: VOLUNTEER, FAMILY',
  username VARCHAR(100) NOT NULL COMMENT '用户名',
  phone VARCHAR(20) COMMENT '手机号',
  appeal_type VARCHAR(50) COMMENT '申诉类型',
  description LONGTEXT COMMENT '详细描述',
  attachment VARCHAR(500) COMMENT '附件',
  status VARCHAR(50) DEFAULT 'PENDING' COMMENT '申诉状态: PENDING, PROCESSING, RESOLVED, REJECTED',
  response LONGTEXT COMMENT '处理回复',
  responded_by BIGINT COMMENT '处理人ID',
  responded_at DATETIME COMMENT '处理时间',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  KEY idx_user_id (user_id),
  KEY idx_status (status),
  KEY idx_user_role (user_role),
  KEY idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户申诉表';
```

## 功能流程说明

### 申诉提交流程
1. 用户访问申诉页面（volunteer_appeal.jsp 或 family_appeal.jsp）
2. 如果已登录，用户名和手机号自动填充；未登录需手动输入
3. 用户选择申诉类型、输入详细描述
4. 点击提交，POST 到 /user/[role]/submit-appeal
5. 控制器调用 AppealService.submit()
6. 申诉保存到 t_appeal 表，状态为 PENDING
7. 用户被重定向到登录页面，显示成功消息

### 申诉审核流程（管理员）
1. 管理员登录后访问 /admin/appeal/list
2. 查看申诉列表，可按状态、角色、用户名筛选
3. 点击"查看详情"进入审核页面
4. 填写处理意见，选择新状态（PROCESSING/RESOLVED/REJECTED）
5. 点击提交审核
6. 申诉状态更新，记录处理人 ID 和处理时间

### 密码修改流程
1. 用户访问密码修改页面
2. 输入用户名、手机号、旧密码、新密码
3. 提交表单 POST 到 /user/[role]/update-password
4. 控制器验证：用户存在、手机号匹配
5. 调用 UserService.updatePassword()
6. 旧密码验证成功，新密码 MD5 加密并保存
7. 用户被重定向到登录页面，提示使用新密码登录

## 常见问题解决

### 404 错误 - 申诉管理页面未找到
- 确保已执行 Maven clean package -DskipTests
- 重启 Tomcat
- 检查 appeal_list.jsp 和 appeal_detail.jsp 是否在 target/community_help/WEB-INF/views/admin/ 目录中

### 数据库错误 - t_appeal 表不存在
- 执行上面提供的 CREATE TABLE SQL 语句
- 验证数据库中表是否已创建：`SHOW TABLES LIKE 't_appeal';`

### 申诉无法提交 - userId 为 NULL
- 确认用户已登录（Session 中存在 CURRENT_USER）
- 检查是否从 Session 中正确获取了用户对象
